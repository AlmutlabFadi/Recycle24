# WORKFLOW_RULES

## الهدف
هذا الملف يحدد طريقة العمل الإلزامية على المشروع لمنع الفوضى والانحدار والتعديلات العشوائية.

---

## 1. قواعد عامة

1. لا يبدأ feature جديد قبل أن تكون قاعدة العمل الحالية مستقرة.
2. كل تعديل يجب أن يكون له هدف واحد واضح.
3. أي إصلاح أو refactor يجب أن يمر عبر تحقق فوري:
   - build
   - grep checks عند الحاجة
   - verification command عند الحاجة
4. لا يتم اعتبار العمل منتهيا بمجرد تعديل الكود بل بعد التحقق.

---

## 2. قاعدة "تغيير واحد = سبب واحد"

في كل دفعة عمل يجب أن يكون لدينا أحد الأنماط التالية فقط:

- إصلاح أمني
- إصلاح أداء
- refactor معماري
- feature جديد
- cleanup / debt reduction

دمج أكثر من نوع بلا ضرورة مرفوض.

---

## 3. قاعدة التنفيذ

أي تنفيذ يجب أن يمر بهذا التسلسل:

1. قراءة الحالة الحالية
2. تحديد الهدف
3. أخذ backup عند الحاجة
4. تنفيذ التعديل
5. تشغيل أوامر التحقق
6. فحص النتائج
7. تثبيت القرار

---

## 4. قاعدة التحقق الإلزامي

لا يعتمد أي تعديل قبل تحقق مناسب من نوعه:

### تغييرات auth / RBAC / middleware
- grep checks
- build
- verification flow

### تغييرات DB / Prisma
- prisma generate
- build
- query/usage sanity check

### تغييرات routes / APIs
- build
- endpoint grep checks
- smoke verification عند الحاجة

### تغييرات UI فقط
- build على الأقل

---

## 5. قواعد git التشغيلية

1. لا تترك تغييرات غير مفهومة في working tree.
2. لا تجمع cleanup عشوائي مع تعديل منطقي حساس.
3. قبل أي commit يجب معرفة:
   - ما الذي تغير
   - لماذا تغير
   - ما الذي تم التحقق منه

---

## 6. قواعد PowerShell داخل هذا المشروع

1. التنفيذ المفضل يكون بأوامر قابلة للتكرار.
2. أي كتابة ملفات حساسة يجب أن تتبعها قراءة أو grep أو build.
3. لا يفترض نجاح أي تعديل حتى يظهر الدليل.
4. إذا فشل here-string طويل أو PSReadLine نستخدم ملفات أصغر أو أوامر موجهة للملف المستهدف فقط.

---

## 7. قواعد الإغلاق

لا تعتبر المرحلة مغلقة إلا إذا تحقق:
- الدليل النصي أو التنفيذي
- أوامر التحقق
- عدم وجود بقايا قديمة واضحة
- build ناجح إن كان التغيير يمس runtime

---

## 8. ممنوعات

ممنوع:
- "نصلح لاحقا" بدون تسجيل debt
- "غالبا يعمل"
- "يبدو صحيحا"
- تخمين حالة النظام بدون grep / read / build / verify
- تغيير واسع غير قابل للتراجع
